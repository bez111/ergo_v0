## Задачи и критерии

### 1. **Ошибки и риски (blocking / high / medium)**

* **Компиляция/типизация:** TS-ошибки, нарушения strict mode.
* **Рантайм-ошибки:** обращение к `window`/`document` в RSC, неправильные пути импорта.
* **React:** ключи списка, нарушенные зависимости эффектов, утечки, двойной рендер, hydration mismatch.
* **Next.js:** смешение RSC/Client, избыточный `use client`, неправильный `fetch`/кеш, неправильные `metadata`, некорректный `dynamic`.
* **SEO/разметка:** битая JSON-LD, каноникал, prev/next, дубли title/description, оголённые UTM в canonical.
* **Безопасность:** XSS в dangerouslySetInnerHTML, небезопасные источники скриптов, отсутствие валидации данных, секреты в коде.
* **Производительность:** «толстые» клиентские бандлы, неиспользуемые зависимости, синхронные тяжёлые операции, блокирующие импорт-цепочки.

### 2. **Мёртвый и неиспользуемый код**

* Неиспользуемые файлы/экспорты/импорты, unreachable branches, legacy-компоненты.
* Дублирующиеся утилиты и типы.

### 3. **Структурные улучшения**

* Разделение на Server/Client компоненты, code-splitting, динамические импорты.
* Консистентные соглашения (импорт алиасы, именование, структура папок).
* Минимальный размер критического пути (LCP-кандидаты без лишней гидратации).

### 4. **Выходные артефакты**

* Отчёт: «что и где» + **точные пути файлов и строки**.
* **Мини-диффы** (или псевдокод) для фиксов.
* **План задач** с приоритетами (Blocking/High/Med/Low) и оценкой влияния.

## Процесс аудита (по шагам)

### **Фаза 0 — Сборка контекста**

* Определи версию Next.js/TS/React, конфиги `tsconfig.json`, `next.config.js`, `eslint`, `tailwind.config.js`.

### **Фаза 1 — Статический анализ**

* **TS строгий:** включены ли `strict`, `noUncheckedIndexedAccess`, `noImplicitOverride`, `exactOptionalPropertyTypes`.
* **ESLint:** найти **ошибки/варнинги**, включая:
  * `unused-vars`, `no-unused-expressions`, `no-unsafe-optional-chaining`, `no-floating-promises` (если есть), `react-hooks/exhaustive-deps`, `@next/next/no-img-element`, `@next/next/no-html-link-for-pages`, `import/no-cycle`, `import/no-extraneous-dependencies`.
* Отметь правила, которых не хватает (предложи конфиг).

### **Фаза 2 — Поиск «мертвого» кода**

* Неиспользуемые файлы/экспорты/импорты: указать список.
* Дубли функций/типов; старые компоненты, не подключённые к маршрутам.
* Ветки `if/else` недостижимы, дефолт не обрабатывается.

### **Фаза 3 — React/Next-специфика**

* `use client` только там, где реально нужен интерактив.
* Проверить ключи в `.map()`, стабилизацию пропов, мемоизацию горячих списков.
* Гидратация: отсутствие различий SSR/CSR (дата/часовой пояс, `Math.random`, `Date.now`, `window`).
* `next/image`: **обязательные** width/height или `sizes`/`fill` + отсутствие CLS.
* RSC-границы: никакого обращения к браузерным API в серверных компонентах.
* `Link`: корректный `prefetch`, отсутствие `<a>` голым без `Link` там, где нужен роутинг.

### **Фаза 4 — Безопасность**

* Любой `dangerouslySetInnerHTML` — проверить санитацию.
* Внешние скрипты → через `next/script` с `strategy`.
* Проверить, что `process.env.*` не утекает в клиент без `NEXT_PUBLIC_`.
* Валидация входных данных в API-роутах/серверных экшенах.

### **Фаза 5 — Производительность**

* Выявить тяжёлые зависимости в клиенте (date-fns/lodash/markdown-парсеры и т.п.).
* Предложить `next/dynamic` для не-Above-the-Fold.
* Удалить дубли импортов, общий код вынести в RSC.
* Проверить `fetch` (кеш/тег/реализация), избегать N+1 запросов.
* Стабильные размеры карточек/героя, отсутствие анимаций до гидратации.

### **Фаза 6 — SEO/Разметка**

* JSON-LD валиден: корректные типы, `@id`, `mainEntityOfPage`, ISO-даты.
* Мета-теги (title/description), canonical, prev/next, `robots`.
* Нет дублей контента (переезды, canonical на page=1), нет index bloat.

## Чек-лист обнаружения типичных проблем (пройди пункт за пунктом)

- [ ] **TS:** неиспользуемые типы/экспорты; `any`/`unknown` без явных сужений.
- [ ] **React:** неправильные `key`, устаревшие refs, лишние эффекты, неочищенные таймеры/слушатели.
- [ ] **RSC/Client:** утечки `use client`, браузерные API на сервере, глобальные синглтоны.
- [ ] **Next:** некорректные `redirect/notFound` в серверных функций, неправильный `revalidate`, дубли `generateMetadata`.
- [ ] **`next/image`:** нет размеров → CLS; `priority`/`fetchPriority` у LCP; корректный `sizes`.
- [ ] **Импорты:** цикличность, избыточные barrel-файлы, относительные/абсолютные алиасы.
- [ ] **Безопасность:** `dangerouslySetInnerHTML` без санитайзера, утечка секретов, открытые CORS.
- [ ] **JSON-LD:** структура и типы, ISO-даты, без лишних полей/дубликатов скриптов.
- [ ] **Мёртвый код:** файлы/экспорты/импорты не используются.
- [ ] **Бандлы:** крупные клиентские зависимости — candidates для `dynamic`/RSC.
